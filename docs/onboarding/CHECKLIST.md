# Onboarding Checklist

Use this checklist to track your progress during onboarding. Mark items as
complete as you go!

## Pre-Start (Before Day 1)

- [ ] Receive offer letter and sign
- [ ] Complete HR paperwork
- [ ] Receive laptop and equipment
- [ ] Receive GitHub account invitation
- [ ] Receive Slack workspace invitation
- [ ] Receive email account setup
- [ ] Review welcome email with start date/time

## Day 1: Welcome & Setup

### Morning

- [ ] Arrive and meet the team
- [ ] Complete IT setup
- [ ] Set up email and calendar
- [ ] Join Slack workspace
- [ ] Introduce yourself in `#general`
- [ ] Meet with manager (1:1)
- [ ] Meet your assigned mentor
- [ ] Review team structure and org chart

### Afternoon

- [ ] Read [Welcome Guide](WELCOME.md)
- [ ] Join all relevant Slack channels
- [ ] Set up GitHub account and SSH keys
- [ ] Request access to required tools
- [ ] Review project README
- [ ] Start [Development Environment Setup](SETUP.md)

### End of Day 1

- [ ] Development environment partially set up
- [ ] Slack profile updated with photo and bio
- [ ] Calendar synced with team meetings
- [ ] Questions list started for tomorrow

## Day 2: Learn the System

### Morning

- [ ] Complete [Development Environment Setup](SETUP.md)
- [ ] Clone repository successfully
- [ ] Install all dependencies (`pnpm install`)
- [ ] Run tests successfully (`pnpm test`)
- [ ] Start development server (`pnpm dev`)
- [ ] Verify API health check endpoint

### Afternoon

- [ ] Read [Architecture Overview](ARCHITECTURE.md)
- [ ] Complete [Codebase Tour](CODEBASE_TOUR.md)
- [ ] Explore packages directory
- [ ] Run `pnpm build:all` successfully
- [ ] Review recent PRs on GitHub
- [ ] Review open issues labeled "good first issue"

### End of Day 2

- [ ] Full development environment working
- [ ] Understanding of system architecture
- [ ] Identified potential first issue to work on
- [ ] Reviewed with mentor

## Day 3: First Contribution

### Morning

- [ ] Read [Development Workflow](WORKFLOW.md)
- [ ] Read [Code Review Guidelines](CODE_REVIEW.md)
- [ ] Pick a "good first issue" from GitHub
- [ ] Assign issue to yourself
- [ ] Create feature branch
- [ ] Set up VS Code with recommended extensions

### Afternoon

- [ ] Write tests for your first change (TDD)
- [ ] Implement your first code change
- [ ] Run tests locally (all passing)
- [ ] Run linter (`pnpm lint`)
- [ ] Run type checker (`pnpm typecheck`)
- [ ] Format code (`pnpm format`)

### End of Day 3

- [ ] First code change committed
- [ ] First PR submitted
- [ ] PR description follows template
- [ ] Tests passing in CI
- [ ] Requested review from mentor

## Day 4: Deep Dive

### Morning

- [ ] Address PR feedback (if any)
- [ ] Participate in stand-up meeting
- [ ] Review [Testing Guide](TESTING.md)
- [ ] Explore test files in codebase
- [ ] Review [Debugging Guide](DEBUGGING.md)

### Afternoon

- [ ] Complete
      [Tutorial 1: First API Endpoint](tutorials/01-first-api-endpoint.md)
- [ ] Explore monitoring dashboard
- [ ] Review deployment documentation
- [ ] Set up local Docker services
- [ ] Test message queue locally

### End of Day 4

- [ ] First PR approved (or in review)
- [ ] Completed at least one tutorial
- [ ] Understanding of testing strategy
- [ ] Familiar with debugging tools

### Morning

- [ ] Get first PR merged! ðŸŽ‰
- [ ] Pick second issue to work on
- [ ] Attend Friday demo meeting
- [ ] Share what you learned this week

### Afternoon

- [ ] Start working on second PR
- [ ] Review a teammate's PR
- [ ] Update documentation (if needed)
- [ ] Schedule 1:1s with team members
- [ ] Reflect on first week

### End of Day 5 (First Week Complete!)

- [ ] At least 1 PR merged
- [ ] Participated in code review
- [ ] Attended all team meetings
- [ ] Development environment fully working
- [ ] Comfortable with development workflow

## Week 2: Build Momentum

### Goals

- [ ] Complete 2-3 PRs
- [ ] Review at least 2 PRs from teammates
- [ ] Start working on a feature (not just bugs)
- [ ] Attend tech review meeting
- [ ] Participate actively in stand-ups
- [ ] Share learning in team channel

### Technical

- [ ] Read [API Development Guide](API_DEVELOPMENT.md)
- [ ] Read [Monitoring Guide](MONITORING.md)
- [ ] Complete [Tutorial 2: Add AI Provider](tutorials/02-add-ai-provider.md)

- [ ] Explore Sentry error tracking

### Team Integration

- [ ] 1:1 meetings with 3+ team members
- [ ] Participate in tech discussion
- [ ] Ask questions in team channels
- [ ] Help another team member
- [ ] Share a tip or trick you learned

## Week 3-4: Feature Development

### Goals

- [ ] Pick up a medium-sized feature
- [ ] Write design doc or RFC (if applicable)
- [ ] Implement feature with comprehensive tests
- [ ] Update API documentation
- [ ] Get feature merged

### Technical

      [Tutorial 3: Monitoring Dashboard](tutorials/03-create-monitoring-dashboard.md)

- [ ] Understand caching strategy

### Contributions

- [ ] At least 5 code reviews given
- [ ] Documentation contribution
- [ ] Bug fix or performance improvement
